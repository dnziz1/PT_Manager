Kevin Hills - Project Breakdown For ersonal Trainer Manager Application


JUMP TO:   Login   Profile   Calendar   Programs   Classes   Payments   Chat   Notifications

CODING - The app will be written mainly using Java programming language and will be adapted to run on mobile phones using Android Studio.
DATABASE - MySQL has been chosen as the backend database which can normally be connected to using the JDBC API/driver in Java.
For this project the MySQL database will reside on a localhost server. In the real world, the database would reside on a remote server.
We will use the Rest api to connect to the database from Android Studio

FEATURE LIST (BASIC):

1. Client and trainer role login (User Profile/account creation)
2. Profile - Search, display, edit
3. Calendar - Showing list of booked classes and other events
4. Programs - Created by trainers containing workout exercises to achieve a specific goal over a number of days
5. Classes - Created by trainers which can be booked by a client.
6. Payments - Simulated payment system for clients to pay for classes and subscriptions
7. Chat - Single or group messaging (public or private)
8. Notifications - Showing recent activity

FEATURE LIST (DETAILED):

1. LOGIN & USER PROFILE CREATION - Login screen where a username and password is required.

Standard username and password fields.
Username constraints - 6 to 30 characters seems to be the accepted standard. Will consist of a combination of alphabetic and numeric characters.
Password constraints - The standard minimum length is usually 8 characters with a maximum length of at least 64 characters.
Password to consist of at least one special character, plus at least one uppercase character plus at least one numerical character.
Passwords will be stored securely using SHA2 256bit encryption.
The constraints must be shown on screen to assist the user when choosing a password.

The Login screen will have a Register option which will then display a user profile creation screen.
The users roleID will determine which features a user can access.
A trainer will have extra fields to store their certifications and specialities
A username will be setup for system administrators so that all featurs are accessible for testing and admin/audit purposes to allow full use of the system without having to go into the backend database.
Perhaps as a future enhancement - add two factor authentication to verify the user by sending an authorisation prompt on their mobile phone.

Database requirements:
User table to hold all generic user details (username, password, first name, surname, display name, date of birth, phone no, email, address including country, personal webpage url, profile picture link, role, notes)
Roles table to hold a list of roles eg client/trainer/sysadmin linked to the user table
Trainer table to hold specific info just for trainers (linked to the User table):
Certifications field (a list of certifications)
Specialities field (a list of specialities)

2. PROFILE - By default a screen is displayed showing the currents users profile.

There is also an option to edit the users own profile
There is an option to search for another profile to display based on textual filter criteria.
When viewing another profile there is an option to view their calendar.

3. CALENDAR - A screen which displays a calendar showing timeslots with the users events/classes and available/unavailable time slots

An option for trainers to add availability - Select days of the week and 1 or more timeslots per selected day
An option to edit/delete a class or booking which would send a notification to all affected users. Only trainers can delete a class.
Selecting a blank timeslot allows a new event/class to be entered
Adding a new event/class - Enter title, description, start date, start & end times, repeat option, event type, client/s, reminder option, notification type
Clients have an option to search for a specific trainers calendar.
After selecting a class timeslot, details of the event will be displayed with an option to make a booking request.
The trainer has to confirm the booking which will raise an outstanding payment request.


Database Requirements:
Schedule table (linked to client and trainer in the user table plus a link to the Classes table) - contains a list of scheduled classes/events
A field for the Name of the date & time of the event, along with a fields that describes the event if it is not a class

4. PROGRAMS - By default a screen lists all programs in alphabetical order but can be also sorted by the most recent.

Search program option - Search box to enter text criteria which will then display a list of matching programs.
After selecting a program a screen is displayed showing all the programs info along with a link to the trainers profile
TRAINER SPECIFIC -
Create option displays a screen allowing you to choose the program name, description plus a box to enter other useful information.
The screen will also have a flexible table where you can enter 1 or more rows of workout days.
Each row of the table will have a day number and a large box to enter the workout/exercise info
Edit option allows a trainer to amend one of their programs - It displays a screen allowing all fields to be changed.
Delete option which allows a trainer to delete any of their own programs.


Database Requirements:
Programs table (linked to a trainer in the user table) - contains a list of training programs created by the trainer
A field for the Name of the program, along with a field that describes the workouts involved and any other useful info such as food/nutrition
Program Days table (linked to the Programs table) - contains a list of workout/exercise info for a specific day in the program

5. CLASSES - By default a screen lists all classes in alphabetical order but can also be sorted by the most recent

Search class option - Search box to enter text criteria which will then display a list of matching classes.
After selecting a class a screen is displayed showing all the class info with a link to the trainers profile
TRAINER SPECIFIC -
Create option displays a screen allowing you to choose the class name, description plus other useful information. There will also be a link to a personal trainer profile.
Edit option allows a trainer to amend one of their classes - It displays a screen allowing all fields to be changed.
Delete option which allows a trainer to delete any of their own classes.


Database Requirements:
Classes table (linked to a trainer in the user table) - contains a list of classes created by the trainer
A field for the Name of the class, along with a field that describes the class and any other useful info such as normal days and times plus who it is suitable for

6. PAYMENTS - A screen is displayed showing a history of payments received/made,

Outstanding payments, Send a payment option which allows a user to enter card details

Database Requirements:
A table to hold a list of payments made/received linked to a client or trainer in the User table

7. CHAT - An icon that is clickable which displays the group chat screen.

Search chat option - Search box to enter search text which then displays a list of matching group chats.
Create a group chat room (0 to many clients) - Enter room name, Option to add clients, option to choose public or private visibility.
Edit group chat room - Amend room name or clients, Delete room
Created chat rooms are added to a list which can be sorted alphabetically or by newest first.
After selecting a group chat room a list of messages are displayed with a box underneath to enter a new message. The messages can be sorted alphabetically or by most recent first.
A user has the option to delete any of their messages but the message will be hidden on screen and stored for completeness and investigation purposes
A notification is sent to the all active members when there is a new message to read.

Database Requirements:
A table to hold a list of all the group chat rooms
A table to hold a list of all messages linked to a group chat room

8. NOTIFICATIONS - An Icon that is clickable which displays a screen showing all recent notifications (latest first).

If appropriate a link is given which links to the appropriate feature/chat message etc

Database Requirements:
A table to hold a list of all notifications linked to a user, ordered by newest date first
